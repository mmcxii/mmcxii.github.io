About Me

Hello! My name is Nich Secord, I am a front-end leaning full stack web developer and designer (entering the workforce no later than November 2019) based in the Seattle, Washigton area. I enjoy creating beautiful and responsive layouts that look great on any screen large, small, and every size in between. I am a proponent of using semantic HTML in tandem with CSS Grid and Flexbox to create accesible sites for those that are incapable of viewing the page as well. I am firm in this belief because as stewards of the front-end it is fundamental to our job to ensure that the internet be easy to use for everyone, regardless of ability.

When away from the office I love spending time outdoors and/ or with friends. I am always looking for another opportunity to go skiing, hiking, camping, or golfing; or for a more leisurly time I enjoy a wide variety of beer, wine, and spirits, as well as pairing cuisine to accompany them. For my money there's no better poison than a Bourbon Manhattan with a splash of Grand Marnier!

Sass

Sass is potentially my favorite individual piece of my workflow. When I first learned Sass is when I really fell in love with front end development, it opened my eyes to how fun this business can be. Functions, variables, and mixins have drastically incresed my productivity and I can't imagine writing CSS without them. I have recently began to incorporate the BEM Naming Convention into my classes, which when combined with nesting makes writing carefully scoped CSS feel incredibly natural and easy.

Javascript

Javascript brings life to static pages in ways that cannot be understated. On this page I am using JS to lazy load images and power these modals, but that is hardly a fraction of what the language is capable of. I also use Javascript to make API calls to send and retrieve data, and then dynamically render that data on the page. I love programming in Javascript and I am always learning new ways to incorporate this wonderfully powerful tool into my projects.

Git

Git is in many ways the most important technology available to developers. Having such granular version control is a very powerful tool, and it makes collaboration on large projects possible with minimal headaches. I feel that a strong understanding of Git is an essential trait in a developer and as such I practice with Git as often as possible to maintain that strict standard.

Node.js

Node.js is the newest addition to my repertoire, as I begin to expand my knowledge of back end technologies. Allowing Javascript to run on the server makes development feel much more fluid as I don't need to know any additional sytax to get my servers up and running. I have some experience with C# which I feel gives me a bit of an advatage when writing in Node. It feels like striking up a conversation with an old friend. I look forward to becoming stronger with this technology and expanding my control over the full stack of my development.

My Work

Over the course of my career these projects stand out to me as examples of my best work. The designs are some of my personal favorites and I am proud of the functionality each one has. Click on a project's name to see it live, or click on the Github icon to see the its repositrory.

The Reactulator

This is the first project I made on my own using React. It is a simple calculator, it works as you would expect. The calculator itself is built using CSS Grid, and the rest of the page relies on a little bit of Flexbox. I really enjoyed putting this very simple project together, React's class-based components feel natural and easy to use, and the fact that using Sass with React involves no additional hoops to jump through almost makes it feel like I'm cheating somehow. I look forward to building more complex applications with these powerful tools.

"The Reactulator" Github Repository

Let's Taco Bout It

This was the first group project I participated in during my Boot Camp in 2019. I collaborated on the front-end, creating uniform components to be used across the site. We agreed on a mobile first approach to ensure that the site would look good on screens of all sizes, and I helped my fellow front-end collaborator to learn and use the Sass CSS pre-processor. We agreed to use the BEM class naming convetion to remove the overhead of deciding what to call certain elements, as well as for its ease of use with Sass's nesting functionality.

As this was my first collaborative project it was also my introduction to Git and Github's tools for working with teammates such as Branching and Pull Requests. I greatly enjoyed working with my team; we communicated well and took great care in planning who was responsible for which parts of the documents. Thanks to our careful planning we had less than five total merge conflicts and the project was a true joy to work on.

Though I did not play a role in creating the back-end for this project I did help select the API's that our team used: the Taco Fancy API, and the Yelp API.

"Let's Taco Bout It" Github Repository

RPS: The Gathering

This game was my first interaction with any sort of database, in this case Google Firebase. I found this project extremely challenging and it took my longer than any assignment throughout the Boot Camp to this point. To be precise, I had troubles grasping how I was to structure my database calls in conjunction with handling which user was which, and keeping track of users when one would leave the lobby. I am very pleased with how well the app came together in the end after how long I struggled with getting it to function.

The theme is based entirely on the card game, Magic: the Gathering, a strategy game my friends and I have enjoyed for a decade now. This is one of my favorite designs I have executed in my career; in particular I love the way the buttons change when one is hovered, and how the content cards change color every time the page is loaded. It is a simple design to be sure but the look of the project came together exactly how I pictured it in my mind.
